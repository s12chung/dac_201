Most people associate typography with fonts. These slides will go into an in-depth explanation on how to select fonts for a website.

Serifs

Before going into the various types of fonts, you should understand that most fonts can generally fall under two categories, serif and sans-serif.

  • Serif fonts have non-structural details at the ends of strokes on some letters called serifs (highlighted in red below)
  • Sans-serif fonts don't have serifs ("sans-serif" literally means "without serif" in French)

Serif fonts are easier to read because the serifs help the words flow and distinguish differences between characters. They have been very popular with print copies.

Sans-serif fonts more popular on computers because low resolution screens and slow computers make difficult to show the serifs and thinner strokes of serif fonts on the computer screen, especially for small font sizes.

In terms of style, serif fonts are look for traditional, while san-serif fonts are clean and modern.

As a general rule, it's best to stick with two different fonts, one serif and one sans-serif (unless you're a designer) because it is often difficult to tell the difference between two of the same type of fonts. Also, it's common to see the less readable font as a header and the more readable font as the body because of the different font sizes.

Statistically, 60% of web sites use sans-serif fonts for headlines, and 66% of web sites use sans-serif fonts for the body. Arial, Georgia and Verdana are the most popular fonts. See this link for more statistics.

Web fonts and font stacks

Generally, in order for web sites to display a font, the web site viewer must have the font installed on their computer. Otherwise, the web browser will show the default serif or sans-serif font installed.

Sadly, not everyone has all the fonts installed. And different operating systems (i.e. Mac or Windows) have different pre-installed fonts. For example, the 99.71% of Mac computers have Helvetica (which is considered the Apple font), compared to less than 10% of Windows computers (Arial is considered Windows' Helvetica).

To compensate for this problem, web designers create font stacks. A font stack is a prioritized list of fonts, where the browser will go through until it finds a font that is installed in the computer, so the can be displayed.

Implementation

Implementing font stacks is simple. Just add the CSS property, font-family.

Below is an example of a font stack:

body { /* means for all text */
   font-family: "Meta Serif Bold", "Lucida Bright", Cambria, Georgia, serif; /* Use quotes for font names longer than one word */
}

which means check the computer for Meta Serif Bold and if it's not installed, try Lucida Bright, then Cambria, Georgia, and then default serif font. An image of the fonts is shown below.
