Here's a quick summary of what was presented.

Font

Quick notes:

  • There are two types of fonts, serif and sans-serif fonts
  • Sans-serif fonts are easier to read on the computer
  • Fonts need to be installed on the user's computer to be displayed, so font stacks compensate for this problem

To select a font:

  • It's best to stick with two different fonts, one serif and one sans-serif
  • Arial, Georgia and Verdana are the most popular fonts
  • This web page has more recommended font stacks and font pre-installation statistics
  • Use implement font stacks, use the font-family CSS property

Size

About size:

  • Headlines should be about 18 to 26 pixels
  • Body copy 12 to 14 pixels
  • Use the font-size CSS property

And for weight:

  • Bolding is preferred to emphasize
  • Limiting opacity is common to de-emphasize text
  • Limit combining and avoid overdoing weighting methods

The code below is CSS implementation of weight:

.some_class {
    font-weight: bold;
    text-decoration: underline;
    font-style: italic;
    font-variant:small-caps;
    text-transform: uppercase;
    color: #222222; /* this is dark gray */
}

Space

Quick notes about measure:

  • Measure is the width of the body, which should be about 55 to 85 characters in length (including spaces)
  • Implementation of measure depends on the situation, try following a grid system like the 960 grid system

And about leading:

  • Leading is the vertical space between lines of text
  • Leading should be wider than word spacing
  • The average line height to body copy font ratio should be 1.5
  • Use the line-height CSS property to implement leading

Other spacing notes:

  • The average ratio between measure and leading is 27.8, which is fairly large
  • The space between paragraphs is about 75% of line height, and use the top-margin property on p tags to implement this

Hierarchy

Just remember to consider how text stands out to the whole design and compare that to the text content's importance to the user and their situation.
